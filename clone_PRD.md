# 📋 PRD (Product Requirements Document) — GlobalMeet

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | GlobalMeet |
| **버전** | 1.0.0 |
| **목적** | 기존 PromiseU의 "글로벌 미팅" 기능을 독립 웹사이트로 분리·클로닝 |
| **대상 사용자** | Google 계정을 보유한 전 세계 사용자 |
| **플랫폼** | 반응형 웹 (데스크톱/모바일) |
| **배포** | Vercel 또는 Firebase Hosting |

---

## 2. 사용자 페르소나

### 페르소나 A: 글로벌 팀 멤버
- Google 계정으로 간편 로그인
- 자신의 위치를 지도에 핀으로 등록
- 팀원들의 위치를 한눈에 확인
- 실시간 채팅으로 소통

### 페르소나 B: 이벤트 주최자
- 여러 장소를 지도에 등록하고 공유
- 대화방을 만들어 참가자를 초대
- 실시간 위치 공유로 이동 중인 참가자 확인

## 4. 사용자 시나리오

### 시나리오 1: 첫 방문 — 핀 등록
```
1. 사용자가 사이트 접속
2. /login 페이지에서 "Google로 로그인" 클릭
3. Google 계정 선택 → 팝업 인증
4. 메인 페이지 (/) 로 리다이렉트
5. 지도 상단 입력란에 "서울특별시 강남구 테헤란로 152" 입력
6. "핀 추가" 클릭 → Nominatim geocode → 지도에 빨강 핀 표시
7. 지도가 해당 위치로 부드럽게 이동 (flyTo)
```

### 시나리오 2: 지도 클릭으로 핀 추가
```
1. 사용자가 지도의 원하는 위치를 클릭
2. 오렌지색 임시 핀이 나타남
3. 주소가 자동 확인됨 (reverse geocode)
4. "이 위치에 핀을 추가할까요?" 카드 표시
5. 핀 제목(선택) 입력 후 "핀 추가" 클릭
6. 핀이 빨강으로 변경되어 저장됨
```

### 시나리오 3: 대화방 만들기
```
1. 지도 하단의 채팅 영역에서 "새 대화방" 클릭
2. 대화방 이름 입력 (예: "주말 모임")
3. 참여자 검색 → 사용자 선택 (최소 1명)
4. "만들기" → Firestore에 대화방 생성
5. 메시지 입력 → 실시간으로 상대방에게 전달
```

### 시나리오 4: 클러스터링 확인
```
1. 다수의 핀이 등록된 상태에서 줌아웃
2. 인접한 핀들이 초록/노란색 숫자 뱃지로 묶임
3. 클러스터 클릭 → 해당 영역으로 줌인 → 개별 핀 표시
4. 다시 줌아웃하면 클러스터로 재묶임
```

## 5. 비기능 요구사항

| 분류 | 요구사항 |
|------|----------|
| **성능** | 핀 1000개 이상에서도 클러스터링으로 지도 렌더링 부드러움 유지 |
| **반응성** | 모바일에서 지도 터치 조작 및 채팅 레이아웃 대응 |
| **접근성** | 키보드 내비게이션, aria-label 적용 |
| **보안** | Firestore Security Rules로 인증된 사용자만 읽기/쓰기 가능 |
| **다국어** | 한국어(기본), 영어, 중국어 지원 |
| **호환성** | Chrome, Safari, Firefox, Edge 최신 2버전 |
